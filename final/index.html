Game Development Addendum

As discussed, there are many crucial steps in publishing a “Lights Out” game using HTML, CSS, and JavaScript. Below, we describe the development process:

HTML Structure

The game board is created using <div> elements to represent the squares. These elements are dynamically added to the page using JavaScript.

CSS Grid Layout

To position the squares in a grid and define their properties, CSS Grid was utilized. This ensures a responsive and visually appealing layout for the game board.

JavaScript Functionality

  • Toggle Mechanism: JavaScript handles the toggle behavior of squares, switching between "on" and "off" states when a square is clicked.
  • Neighbor Logic: Clicking a square also toggles the adjacent squares (top, bottom, left, and right).
  • Victory Condition: The game checks whether all squares are "off" at the end of each action. If so, the player wins, and a congratulatory message is displayed.

Additional Features

Several features were considered to enhance the gameplay experience:

  • Restart Button: A button to restart the game from the beginning, allowing users to start a new challenge.
  • Move Counter: Tracks the number of actions taken by the player, encouraging efficiency.
  • Timer: Adds a time-based challenge to make the game more competitive and engaging.

Testing and Responsiveness

The game was tested across multiple browsers to ensure compatibility and responsiveness. It was designed to work seamlessly on desktop and mobile devices.

User Experience

For users interested in casual games, the focus was on creating an enjoyable and straightforward gaming experience. Feedback and suggestions from players were used to address any issues and improve the overall design.
